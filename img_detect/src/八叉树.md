实现 **八叉树地图 (Octomap)** 不需要你从零开始写 C++ 数据结构代码。在 ROS 生态中，已经有非常成熟的标准包 **`octomap_server`**。

你的任务主要分为三步：**安装包 -> 编写 Launch 文件 -> (可选) 接入 MoveIt**。

以下是针对你的 ZED + Rokae 机械臂项目的具体实现流程。

---

### 1. 核心概念：为什么要用 Octomap？

* **点云 (Point Cloud)**：只是一堆点的集合。由于噪点，它看起来是“虚”的，且无法区分“未知区域”和“空气”。
* **八叉树 (Octomap)**：将空间切分成一个个**小方块 (Voxels)**。
* 它不仅知道哪是个**障碍物 (Occupied)**。
* 它还知道哪里是**空气 (Free)**（光线穿过的地方）。
* 它知道哪里是**未知区域 (Unknown)**（相机没看过的死角）。
* **最重要**：它能把 ZED 相机的噪点滤掉（通过概率更新）。



---

### 2. 第一步：安装依赖

在终端中运行：

```bash
sudo apt update
sudo apt install ros-noetic-octomap-ros ros-noetic-octomap-server

```

*(如果你用的是 Ubuntu 18.04/Melodic，把 `noetic` 换成 `melodic`)*

---

### 3. 第二步：编写 Launch 文件

在你的 `apple_harvesting` 包的 `launch` 文件夹下，新建一个文件 `octomap_mapping.launch`。

**关键点：**

1. **输入话题 (`cloud_in`)**：这里建议直接接入 ZED 的**原始点云**话题（通常是 `/zed2i/zed_node/point_cloud/cloud_registered`），而不是我们之前那个过滤过的 `/ap_robot/obstacles`。
* *为什么？* 因为 Octomap 需要通过光线投射（Ray Casting）来清除旧的障碍物（把以前是有树枝、现在树枝移走的地方变成空气）。如果我们只发障碍物点云，Octomap 就永远无法清除障碍，只会越堆越多。


2. **坐标系 (`frame_id`)**：必须设为世界坐标系（如 `base_link` 或 `world`）。

```xml
<launch>
  <node pkg="octomap_server" type="octomap_server_node" name="octomap_server">
    
    <param name="resolution" value="0.02" />
    
    <param name="frame_id" type="string" value="base_link" />
    
    <param name="sensor_model/max_range" value="2.0" />
    
    <param name="sensor_model/hit" value="0.7" />
    <param name="sensor_model/miss" value="0.4" />
    <param name="sensor_model/min" value="0.12" />
    <param name="sensor_model/max" value="0.97" />
    
    <param name="filter_ground" value="false" />
    
    <remap from="cloud_in" to="/zed2i/zed_node/point_cloud/cloud_registered" />
  
  </node>
</launch>

```

### 4. 第三步：运行与 RViz 可视化

1. 启动你的 ZED 相机节点。
2. 启动 Octomap：
```bash
roslaunch apple_harvesting octomap_mapping.launch

```


3. 打开 RViz：
* **Add** -> **By Topic** -> 找到 `/octomap_point_cloud_centers` -> 添加 **PointCloud2**。
* 或者添加 **MarkerArray** (`/occupied_cells_vis_array`)，这会显示一个个彩色的小方块。
* **Color Transformer**: 改为 `Z Axis`，你会看到非常漂亮的体素地图。

### 5. 第四步：在 C++ 代码中使用 Octomap 做碰撞检测

既然你已经有了 Octomap，你的 `checkCollision` 函数可以升级得非常专业。

我们需要使用 FCL (Flexible Collision Library) 或者直接查询 Octomap 数据结构。这里提供一种**直接查询 Octree** 的轻量级方法。

**在 `smart_robot_node.cpp` 中：**

1. **包含头文件：**
```cpp
#include <octomap/octomap.h>
#include <octomap/OcTree.h>
#include <octomap_msgs/Octomap.h>
#include <octomap_msgs/conversions.h>

```


2. **订阅地图回调：**
```cpp
// 全局变量
std::shared_ptr<octomap::OcTree> g_octree;

// 回调函数：接收 Octomap 二进制流并转为 C++ 对象
void octomapCB(const octomap_msgs::OctomapConstPtr& msg) {
    // 转换 ROS 消息为 Octomap 对象
    octomap::AbstractOcTree* tree = octomap_msgs::msgToMap(*msg);
    if (tree) {
        // 强制转换为 OcTree (最常用的类型)
        g_octree = std::shared_ptr<octomap::OcTree>(dynamic_cast<octomap::OcTree*>(tree));
    }
}

// 在 main 中订阅
ros::Subscriber sub_oct = nh.subscribe("octomap_binary", 1, octomapCB);

```


3. **升级 `checkCollision` 函数：**
现在你可以检测路径上的每一个点是否在方块里。
```cpp
bool checkCollisionOctomap(Vector3d start, Vector3d end, double radius) {
    if (!g_octree) return false;

    Vector3d dir = (end - start).normalized();
    double len = (end - start).norm();
    double step = 0.02; // 检测步长 2cm

    for (double d = 0; d < len; d += step) {
        Vector3d p = start + dir * d;

        // 查询 Octomap 该点是否被占据
        octomap::point3d query(p.x(), p.y(), p.z());
        octomap::OcTreeNode* node = g_octree->search(query);

        if (node && g_octree->isNodeOccupied(node)) {
            // 如果中心点碰到了，那是实打实的碰撞
            return true; 
        }

        // 进阶：如果要做带半径的检测，你需要检查点周围的邻居
        // 或者简单点：如果查询到了，说明离障碍物很近
    }
    return false;
}

```

### 6. (进阶) 如果你想接入 MoveIt!

这是最省事的办法，MoveIt 自带 Octomap 支持。你不需要写上面的 C++ 代码，只需要配置 yaml。

1. 找到你的 MoveIt 配置包（通常叫 `rokae_moveit_config`）。
2. 在 `config` 文件夹下创建或修改 `sensors_3d.yaml`：
```yaml
sensors:
  - sensor_plugin: occupancy_map_monitor/PointCloudOctomapUpdater
    point_cloud_topic: /zed2i/zed_node/point_cloud/cloud_registered
    max_range: 2.0
    point_subsample: 1
    padding_offset: 0.1
    padding_scale: 1.0
    max_update_rate: 1.0
    filtered_cloud_topic: filtered_cloud

```


3. 在 `launch/sensor_manager.launch.xml` 中加载这个文件。
4. **效果**：MoveIt 里的机械臂会自动把 Octomap 当作墙，规划路径时（使用 RRT）会自动绕开这些方块。

### 总结建议

1. 先做 **Step 1 - 3**，把 `octomap_server` 跑起来，在 RViz 里看到彩色的方块地图。这会极大地增强你的演示效果。
2. 由于你现在的控制代码是手写的 `MoveL`，建议配合 **Step 5**，在你的 C++ 代码里订阅 octomap 并查询，这样比之前的点云遍历要快且准。
